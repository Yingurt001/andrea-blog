## 起因

111今天在小红书上刷到一个博主分享她的 Obsidian 主页 —— 粉紫色的渐变背景、翻牌时钟、月历、每日打卡、笔记热力图、本周周记入口。所有的东西都集中在一个看板上，颜色温柔、信息密度刚好。

她在评论区写：「就是 Codex 直接读取本地文件让他帮我修改」。

我当时刚好正在准备搬去剑桥读 DIS（Data Intensive Science）的事情，原本就建了一个空的 vault 文件夹放在 `Academic 学术/University of Cambridge剑桥大学/`，里面只有几份零散的面试材料和住宿指南。我盯着那个文件夹想了一会儿 —— 既然别人能用 AI 搭出这种东西，那我也试一下。

于是有了这篇博客。

## 我让 Claude Code 帮我做了什么

整个过程大约一个半小时，我大部分时间在喝奶茶 + 看截图，剩下的时间在告诉 Claude 哪里不对。

具体做了：

1. 把 Cambridge 那个空文件夹初始化成一个 Obsidian vault（创建 `.obsidian/` 配置）
2. 从 GitHub releases 下载 **Dataview** 和 **Heatmap Calendar** 两个插件
3. 写了一个完全用 DataviewJS 渲染的 `dashboard.md`
4. 写了配套的 CSS snippet，做粉紫风格
5. 配好 Daily / Weekly 的目录结构和模板
6. 修了几个出现的 bug（比如 `this.register is not a function`，比如 Obsidian 默认行宽限制）

最后的成品长这样 —— 顶部是翻牌时钟 + 模拟时钟 + 实时问候 + 三个 chips（连续打卡 N 天 / 本周完成率 / 今日 X/4）；中间一行三栏：月历、每日打卡（运动 / 阅读 / 心情 / 早睡）、笔记热力图；下面一行两栏：本日任务和周记入口。

最让我开心的是它**真的能用**：

- 点击打卡卡片 → 立刻翻面变粉色，daily note 的 frontmatter 会自动写入
- 任务前的 checkbox 打勾 → 文件里的 `- [ ]` 立即变成 `- [x]`
- 月历点任意日期 → 自动打开（或创建）那天的 daily note
- 输入框敲一行字回车 → 直接追加到今日任务

是真正活的看板，不是装饰用的截图。

## 在剑桥我打算用它干什么

这个 vault 的范围是「**所有跟剑桥相关的事**」 —— 从现在的面试准备一路到 Fall 2026 入学之后。

具体会装进去的内容：

**学术侧**

- DIS 的每节课笔记（按 Lecture 编号归档，配合我写过的 `course-pdf` skill 输出 LaTeX 笔记）
- Supervisor meeting 的会议纪要 + action items
- Research project 的进度日志（沿用我 Nottingham 时期的研究日志习惯，写成"今日做了什么 + 卡在哪 + 明天打算"）
- Reading list 和论文精读笔记（配合 `zotero-note` skill 自动从 Zotero 拉全文做笔记）

**生活侧**

- 每日打卡（运动 / 阅读 / 心情 / 早睡）—— 这四项是我观察自己一年下来发现最影响状态的
- 周记 —— 每个周日写一份，回顾这一周做了什么、卡在哪、下周怎么调整
- 本日任务 —— 不写复杂的 todo system，就是 daily note 里一行一行的 checkbox

**仪式侧**

- 每天打开电脑第一件事点开 dashboard，翻牌钟和粉紫渐变会让我从「拖延状态」迅速进入「我今天还有事要做」的状态
- 看到笔记热力图，如果某周一整周是空的，就知道自己摸鱼了，会有羞愧感（这种羞愧感其实挺有用的）
- 月历上点过的日期会有粉点，这种"已经做过的事被可视化"对自律比 todo list 有用得多

## 一些个人想法

### 工具仪式感是真的有用的

我以前一直觉得「折腾工具」是逃避真正的工作 —— 你花一下午配你的 Vim，最后一行代码没写。

但这一两年我改变看法了。**仪式感工具让你愿意打开它**，这一点的价值被低估了。

我以前用过 Notion、Logseq、Apple Notes、纯 markdown + Cursor，每个都试过两三个月。最后留下来的永远是 Obsidian。原因不是 Obsidian 功能最强 —— 它甚至默认体验很丑 —— 而是我会花心思**让它变成我喜欢的样子**。每次打开，看到自己亲手调过的颜色、字体、布局，会有一种「这是我的东西」的归属感。这种归属感是我能坚持记笔记五年的真正原因。

剑桥的一年只有一年。我不想把它过成「随便记几篇笔记，随便上几节课，最后毕业拍个照」。我希望每一节课、每一篇论文、每一次和 supervisor 的会面，都有迹可循。

而要做到这个，我需要一个**让我每天都想打开**的入口。

### Vibe Coding 真的改变了我对"做工具"的预期

如果倒退到三年前，让我从零搭一个这样的 dashboard，我大概会：

1. 花一个礼拜研究 Dataview 文档
2. 在论坛上爬两天别人的代码
3. 一边骂 CSS 一边瞎调三天
4. 第八天放弃，回去用别人现成的模板

但今天是这样的：

1. 我截图给 Claude，说"这个怎么做的"
2. Claude 说先确认你 vault 在哪
3. 我说放剑桥，剩下的你看着办
4. Claude 写完所有代码，连 bug 都自己修了
5. 我只在最后审美的部分介入"这里点击应该 toggle"、"任务卡片要能打勾"、"页面太窄要拓宽"

整个过程我**没有写一行代码**。但成品是 100% 我想要的样子。

这种感觉很难描述 —— 不是"AI 帮我打字"，而是"我变成了一个产品经理，AI 是我的全栈工程师"。我只需要负责审美和判断，剩下的脏活都不用碰。我甚至可以一边做这件事一边喝奶茶看小红书。

我开始相信，未来一两年内，「想要什么样的工具」会比「能做出什么样的工具」重要得多。**审美和判断力会变成稀缺资源**。

### 关于"抄"这件事

我抄了那个小红书博主的设计。我不羞愧。

她在评论里也写了「我借鉴的这个的哒」，指向另一个模板。所有好的设计都是站在前人肩膀上 —— 重要的不是你完全原创，而是你**真的把它跑起来了**，并且改成符合你自己生活的样子。

我把她的运动 / 阅读 / 心情 / 早睡四项打卡照搬，因为这四项也确实是我想跟踪的。但 chips 上面我换成了"连续打卡 / 本周完成率 / 今日完成数"这种我自己关心的指标；任务卡片我加了进度条和实时勾选；月历我加了"点击日期跳转"。

模仿是起点，不是终点。

### 写给未来在剑桥的自己

如果你之后某天打开这个 dashboard，发现：

- 笔记热力图大片空白
- 连续打卡天数停留在个位数
- 本日任务永远是 0/0
- 月历上没几个粉点

不要责怪工具，也不要换工具。**问问自己是不是又在拖延**。

这个 dashboard 是 2026 年 4 月 29 日下午做的，那时候你还在 Nottingham，还没拿到 Cambridge 的 unconditional，还在为 IELTS 和住宿抢位发愁。但你已经决定要把那一年过得认真一点了。

希望未来的你打开这个看板时，看到的是一片粉紫色的小圆点。
